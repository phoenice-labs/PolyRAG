# CONFIDENCE BUG ANALYSIS: Knowledge Graph Results Show 100% Confidence with Low Scores

## ROOT CAUSE IDENTIFIED

The bug is in **/api/routers/search.py** at lines **174** and **255**:

`python
confidence=float(r.score),
`

### Problem Flow:

1. **Graph traversal sets score = 1.0** (/core/graph/traversal.py, line 28)
   - HOP_SCORES = {0: 1.0, 1: 0.6, 2: 0.3}
   - Direct entity matches always get score 1.0
   - This is CORRECT for graph relevance but WRONG for confidence

2. **RRF fusion doesn't normalize graph scores** (/core/retrieval/triple_hybrid.py, lines 164-168)
   - When only graph results are returned (graph-only mode), score stays at 1.0
   - When fused with hybrid results, score is RRF-combined but still treated as final confidence
   - RRF formula: score = 1/(60 + rank) per method, so even rank 1 = 0.0161, not 1.0

3. **Search API passes RRF score directly as confidence** (/api/routers/search.py:174)
   - confidence=float(r.score) — uses the SearchResult.score as confidence
   - This score came from RRF (which is 0.0-0.5 typically), BUT for graph-only or graph-dominant results it can be near 1.0

4. **Frontend multiplies by 100** (/frontend/src/components/ResultCard/ResultCard.tsx:151)
   - {(result.confidence * 100).toFixed(1)}%
   - 1.0 → 100.0%, 0.008 → 0.8% (but 0.008 is being shown as 100%)

## WHY YOU'RE SEEING 100% WITH SCORE 0.00820:

When a graph result ranks very low (e.g., from hop 2 with score 0.3), but is the **only result in graph-only mode**:
- Line 150 sets SearchResult(document=doc, score=score, rank=rank)
- **But the score variable = 1.0 from hop 0 or 0.6 from hop 1**, NOT 0.00820
- The 0.00820 you see is likely from the **metadata field "graph_score"** being displayed elsewhere
- The **confidence field comes from r.score**, which is 1.0 for direct matches
